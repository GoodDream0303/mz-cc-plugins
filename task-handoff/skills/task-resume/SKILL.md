# task-resume — 会话接力

在干净的新会话里读回某个开发任务的交接文档,并从"下一步"继续工作。**不拉取旧对话上下文**
(那会把长历史塞回来,反而更慢)——只读一份精炼的 `handoff.md`。

数据根目录:`${user_config.handoff_root}`,总账为其下 `INDEX.md`。所有文件均为 UTF-8。

要接力的任务名:从用户消息中解析;留空则进入"列表挑选"模式。

## 执行步骤

1. **读总账**:用 Read 读 `${user_config.handoff_root}/INDEX.md`。

2. **未给任务名**:
   - 从 INDEX 里筛出状态为 🔄进行中 / ⬜阻塞 的任务,列成简表(任务名、状态、最近更新)给用户。
   - 请用户指明接力哪一个。**不要自动选**,也不要拉旧对话。停在这里等用户回复。

3. **给了任务名**:
   - 在 INDEX 里按任务名匹配(支持部分匹配/模糊)定位其文件夹列。
   - 命中唯一 → 用 Read 读该文件夹下的 `handoff.md`。
   - 命中多个或零个 → 列出相近候选让用户选,不要猜。

4. **复述确认**:读完 `handoff.md` 后,用三五句话向用户复述:
   - 任务目标(为什么做)
   - 当前进度(做到哪了)
   - 下一步(这次准备做什么)
   - 请用户确认理解一致(若交接文档有"未决问题",一并提出)。

5. **接续**:确认后,直接从 `handoff.md` 的"## 7. 下一步"开始干活,**不要求用户重述背景**。
   过程中遵循该任务记录的关键路径、分支、约束。

> 提示:本会话结束或用户说"准备新开会话"时,用 task-save(`/task-handoff:task-save`)把新进展回写,完成接力闭环。
