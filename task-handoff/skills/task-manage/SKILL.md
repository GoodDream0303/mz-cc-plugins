# task-manage — 接力任务管理

维护数据根目录下的任务总账与任务文件夹。支持四类操作:
**删除 / 改编号重命名改状态 / 归档已完成 / 清理体检**。

数据根:`${user_config.handoff_root}`,总账 `INDEX.md`。所有 md 一律 **UTF-8 无 BOM**。
文件系统操作(移动/重命名/删除文件夹)用 shell;改 INDEX/handoff 内容用 Edit。

## 派发

先用 Read 读 `${user_config.handoff_root}/INDEX.md` 了解现有任务。再解析用户意图:
- 含操作意图(删除/改/归档/清理)→ 直接执行对应小节。
- 不明确 → 列出 INDEX 当前任务 + 下面四个操作菜单,问用户做哪个、对哪个任务。**不要自己猜**。

---

## 1. 删除任务(不可逆,务必谨慎)

1. 在 INDEX 定位该任务的文件夹。
2. **删除前先看目标**:用 Read 看一眼该 `handoff.md`,把任务名/状态/下一步摘要给用户,
   明确告知将**递归删除整个文件夹**(连同里面所有产物),请用户确认。
3. 用户确认后删除该文件夹(Windows PowerShell:`Remove-Item '<文件夹绝对路径>' -Recurse -Force`;
   类 Unix:`rm -rf '<文件夹绝对路径>'`)。
4. 用 Edit 从 `INDEX.md` 删掉对应那一行。
5. 回显:已删除 `<文件夹>` 及 INDEX 中的行。

> 若用户只想删 INDEX 行、保留文件夹(或反之),按其明确意思来,别擅自扩大删除范围。

## 2. 改编号 / 重命名 / 改状态

可单项或组合。改完**三处都要同步**:文件夹名(仅重命名时)、`INDEX.md` 行、`handoff.md` 头部。

- **改编号**:Edit 改 handoff 头部 `编号:` 与 INDEX 的编号列。
- **改状态**:Edit 改 handoff 头部"状态"行与 INDEX 状态列(🔄/⬜/✅),并刷新"最近更新"。
- **重命名任务**(连带文件夹):
  1. 文件夹名规则仍是 `简明任务名_YYYYMMDD`,只换任务名部分、保留原日期。
  2. 重命名文件夹(PowerShell:`Rename-Item '<旧文件夹绝对路径>' '<新文件夹名>'`;类 Unix:`mv`)。
  3. Edit 改 INDEX 的"任务"列与"文件夹"列(`<新文件夹名>/handoff.md`)。
  4. Edit 改 handoff.md 标题行的任务名。
- 回显改了哪些字段、涉及哪些文件。

## 3. 归档已完成

把状态为 ✅已完成 的任务移出主表,保持 INDEX 清爽。

1. 确保 `${user_config.handoff_root}/archive/` 存在(无则新建目录)。
2. 把任务文件夹移入 `${user_config.handoff_root}/archive/`(PowerShell:`Move-Item`;类 Unix:`mv`)。
3. Edit `INDEX.md`:把该行从主表移到文件末尾的 `## 已归档` 区(无该标题则新建),
   文件夹列路径改为 `archive/<文件夹名>/handoff.md`。
4. 可一次处理多个已完成任务;回显归档了哪些。

## 4. 清理示例行 / 体检一致性

1. **清理示例**:删掉 INDEX 里带 `<示例>` 字样的行。
2. **体检**:用 Glob 列出 `${user_config.handoff_root}` 下实际任务文件夹,与 INDEX 行比对,报告两类不一致:
   - **孤儿行**:INDEX 有、磁盘上找不到对应文件夹。
   - **漏登记**:磁盘上有任务文件夹、INDEX 里没有对应行。
3. 只**报告**不一致并给修复建议(补行 / 删孤儿行 / 修路径);**经用户确认后**再改,不擅自增删。
