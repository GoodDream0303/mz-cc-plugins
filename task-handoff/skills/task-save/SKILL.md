# task-save — 会话落盘

把当前会话的进展总结成精简交接文档,写入配置的数据根目录下该任务的独立文件夹,并同步根目录的
`INDEX.md` 总账。目的:让下一个会话能用 task-resume 无损接力,无需用户重述背景。

数据根目录:`${user_config.handoff_root}`(用户安装时配置)。总账为其下 `INDEX.md`。

任务名:从用户消息中解析;留空则按下面第 1 步判定。

## 编码铁律(必须遵守)

数据根下所有 `.md` 一律 **UTF-8**(无 BOM)。这些是给 Claude 工具读写的文档,优先用 Write/Edit
工具(本就按 UTF-8)。若不得已用 PowerShell,须显式 `-Encoding utf8` 且不引入 BOM。

## 执行步骤

1. **判定当前任务**:
   - 若本会话此前已用 task-resume 接力过某任务 → 就是它。
   - 否则若用户给了任务名 → 用它。
   - 否则向用户确认任务名(可据当前 git 分支名推断)。

2. **定位/新建任务文件夹**:根目录为 `${user_config.handoff_root}`。
   - 文件夹名规则:`简明任务名_创建日期`(YYYYMMDD,日期取今天),用下划线不留空格,例 `can总线优化_20260624`。
   - 用 Glob/Read 在 `INDEX.md` 里查该任务是否已有文件夹;有则复用,无则新建文件夹 + 按下方模板新建 `handoff.md`。

3. **写/更新 `handoff.md`**(精简版模板,已存在则用 Edit **只更新变化区块**,不重写无关内容):

   ```markdown
   # <任务名> | 编号:<可选,如任务单号> | 分类:<可选>
   > 状态:🔄进行中 / ⬜阻塞 / ✅已完成
   > 创建:YYYY-MM-DD   最近更新:YYYY-MM-DD
   > 分支:<git 分支名,可选>
   > 关键路径:<SDK / 工程根 / 主要源码 等,可选>

   ## 1. 任务目标        —— 为什么做、做成什么算成功
   ## 3. 当前进度        —— ✅已完成 / 🔄进行中 / ⬜待办
   ## 4. 关键决策与权衡  —— 为什么这么选(避免下个会话重新纠结)
   ## 7. 下一步          —— 新会话第一件事做什么(最重要)
   ## 9. 关键链接与路径  —— 用户提过的网址/工程路径/参考文件,逐字记下,新会话直接用
   - 工程路径:<...>
   - 参考文件:<...>
   - 网址链接:<...>

   <!-- 以下区块按需添加,空着就删掉 -->
   ## 2. 背景与约束      —— 平台/编码/已知坑(一次写清)
   ## 5. 改动清单        —— 涉及文件 + 一句话说明
   ## 6. 验证方法        —— 跑什么、通过标准
   ## 8. 未决问题/备忘
   ```
   - 区块 1/3/4/7/9 必填;2/5/6/8 按需(无内容就删掉该标题)。
   - 每次落盘必刷新"最近更新"日期,并据实更新状态。
   - **下一步**是接力成败关键:写清"新会话第一件事做什么",越具体越好。
   - **区块 9 必做**:回看整段对话,把用户提到过的**网址链接、工程/SDK 路径、参考文件路径、
     命令、关键标识符**等都**逐字**收集进来(不要改写、不要省略路径层级),这些是新会话最贵的上下文。
     沿用时也写进 handoff 头部的"> 关键路径"行。

4. **同步 `INDEX.md`**:根目录 `${user_config.handoff_root}/INDEX.md`。
   - 表格列:`任务 | 状态 | 编号 | 文件夹 | 最近更新`。
   - 已有该任务行 → 用 Edit 更新其状态与"最近更新";没有 → 追加一行,文件夹列写 `<文件夹名>/handoff.md`。
   - 若 `INDEX.md` 不存在,先按此表头新建。

5. **回显**:告知用户已保存到 `<handoff.md 完整路径>`,并提示下次新会话用 task-resume(`/task-handoff:task-resume <任务名>`)接力。
